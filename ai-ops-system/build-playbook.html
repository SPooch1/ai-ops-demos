Pre-Build Setup (Do This for Every Client)
▾

Before you start building anything:

  1. Create a client folder: ai-ops-demos/sites/[client-name]/
  2. Collect all assets from the client onboarding checklist (open onboarding.html)
  3. Save client's logo, headshot(s), and brand colors to the client folder
  4. Create a notes.txt in the client folder with:
    • Business name
    • Industry
    • Packages purchased
    • Key preferences
    • Deadline
  5. If the client purchased a website, start there. The automations reference the website, so build it first.
2
Custom Website Build
▾

What you're building: A single-file HTML website (like the demo sites in the portfolio).

Step 1: Pick a starting template

Open the demo site closest to the client's industry:

  • → CPA / Accounting: sites/cpa/index.html
  • → Dental / Medical: sites/dental/index.html
  • → Law Firm / Professional Services: sites/professional-services/index.html
  • → Manufacturing / Industrial: sites/manufacturing/index.html
  • → General / Other: sites/cpa/index.html (best overall template)

Step 2: Open Claude Code and tell it:

"I need to build a custom website for [Business Name]. They're a [industry] firm with [X] employees. Here's their info: [paste from onboarding checklist]. Use [template file] as the starting point. Brand colors: [X, Y, Z]. Fonts: [X, Y]."

Step 3: Claude Code will:

  • Copy the template to the client folder
  • Replace all placeholder content (business name, team members, services, etc.)
  • Update colors, fonts, and styling to match the client's brand
  • Update contact info, CTAs, and links
  • Add/remove sections based on what the client needs

Step 4: Review the site

Open index.html in your browser (just double-click the file):

  • Check all text for accuracy
  • Verify colors look right
  • Test on mobile (use browser dev tools or shrink the window)
  • Click all links and buttons

Step 5: Client review

Email the HTML file to the client, or host it temporarily on GitHub Pages for preview. Get feedback, make revisions.

Step 6: Deploy to client's hosting

  • SiteGround: Upload via File Manager or FTP
  • Other host: Follow their upload process
  • No hosting: Recommend SiteGround ($3.99/mo starter plan)
  • Point their domain's DNS to the hosting
  • SSL certificate (most hosts do this automatically)
Estimated build time: 2-4 hours with Claude Code
3
New Client Onboarding Automation
▾

What you're building: An automated email sequence that triggers when a new client signs up. Built in MailerLite.

What you need from the client:

  • Welcome message content (or permission to write it)
  • List of onboarding steps/documents the client needs
  • Timeline (how spread out the emails should be)
  • Sender name and email address

Step 1: Log into MailerLite

Use the scott@buildmoremargin.com account, or the client's own account.

Step 2: Create a new Group

Name it [Client Name] - New Clients

Path: Subscribers > Groups > Create Group

Step 3: Create a new Automation

Automations > Create automation > Start from scratch

Trigger: "When a subscriber joins a group" > select the group you just created

Add email steps with delays between them:

Typical 5-email onboarding sequence:
EmailDelaySubjectContent
1ImmediateWelcome to [Business]!Warm welcome, what to expect, first steps
2Day 2Your next step: [Action]First document/form to complete
3Day 5Quick check-inAre they on track? Need help?
4Day 10[Helpful resource]Tips, guide, or FAQ relevant to new clients
5Day 14You're all set!Confirm onboarding complete, introduce ongoing support

Step 4: Write each email

Use MailerLite's drag-and-drop editor:

  • Add the client's logo at the top
  • Use their brand colors for buttons
  • Keep emails short (3-5 paragraphs max)
  • Every email needs a clear CTA button
  • Use {$name} for personalization (MailerLite's merge tag)

Step 5: Create an opt-in form (if needed)

Forms > Create form > Embedded form

  • Style it to match the client's brand
  • Connect it to the "[Client Name] - New Clients" group
  • Get the embed code and add it to the client's website

Step 6: Test the automation

  • Add yourself as a subscriber to the group
  • Verify each email arrives with correct content and timing
  • Check links, images, and formatting
  • Delete yourself from the group when done testing

Step 7: Activate

Activate the automation and show the client how to add new subscribers (or connect their form).

Estimated build time: 2-3 hours

Review Collection Automation
▾

What you're building: An automated sequence that asks happy clients to leave online reviews. Built in MailerLite.

What you need from the client:

  • Which review platforms (Google, Yelp, Facebook, etc.)
  • Direct review links for each platform
  • Timing preference (how long after service)
  • Whether they want a "happy gate" (satisfaction check first)

Step 1: Create a Group

Name it [Client Name] - Review Requests

Step 2: Create the Automation

Without happy gate (simpler):
EmailDelaySubjectContent
1ImmediateHow did we do?"We'd love your feedback. It takes 2 minutes." + direct review link button
2Day 3Quick reminderGentle nudge for those who didn't review
3Day 7Last chanceFinal ask, emphasize how much it helps
With happy gate (recommended):
EmailDelaySubjectContent
1ImmediateHow was your experience?Two buttons: "It was great!" / "Could be better"
  • "It was great!" button links to the review platform
  • "Could be better" button links to a private feedback form (Google Form or email)
  • This way, unhappy clients give feedback privately instead of leaving bad public reviews

Step 3: Get the client's direct review links

  • Google: Search for the business on Google Maps > click "Write a review" > copy that URL
  • Or use a Google Place ID link: https://search.google.com/local/writereview?placeid=[PLACE_ID]
  • Yelp: https://www.yelp.com/writeareview/biz/[BUSINESS_ID]

Step 4: Test, activate, and train the client

Show the client when to add contacts to the group (e.g., after each completed service).

Estimated build time: 1-2 hours

Scheduling & Reminders Automation
▾

What you're building: Automated appointment reminders and follow-ups. Can be built in MailerLite or integrated with their scheduling tool.

What you need from the client:

  • Current scheduling tool (if any)
  • Appointment types
  • Reminder timing preferences
  • Whether they want SMS (requires additional tool like Twilio)

Option A: Email-only reminders (MailerLite)

Best for most clients. Start here.
  1. Create a Group [Client Name] - Appointments
  2. Build automation with reminder emails at chosen intervals
  3. Client manually adds contacts when appointments are booked (or connect via Zapier/n8n if they use Calendly/Acuity)
  4. Post-appointment follow-up email (thank you + review request or next steps)

Option B: Connected to scheduling tool (n8n workflow)

  1. This requires n8n (more advanced)
  2. Webhook trigger from Calendly/Acuity > n8n > MailerLite group
  3. Only pursue this if client specifically needs it. Otherwise Option A works fine.

Option C: SMS reminders (requires Twilio)

  1. Adds complexity and recurring cost ($0.0079/SMS)
  2. Only offer if client specifically requests SMS
  3. Uses n8n: scheduling tool > n8n > Twilio SMS
For most clients, Option A is the right starting point. Keep it simple.
Estimated build time: 1-2 hrs (Option A) | 3-4 hrs (Option B/C)

Report Generator Automation
▾

What you're building: Automated reports compiled from client data. This varies the most by client.

Common approaches:

  • Simple: MailerLite email with formatted data summary (manual data entry)
  • Medium: n8n workflow that pulls from Google Sheets/Airtable and formats into email
  • Advanced: n8n workflow that pulls from multiple sources, generates PDF, and emails it

Simple version (MailerLite)

  1. Create a report template email in MailerLite
  2. Set up a recurring automation (weekly/monthly trigger)
  3. Client updates a shared Google Sheet with their data
  4. Report email summarizes the key metrics

Medium version (n8n)

  1. Create an n8n workflow
  2. Schedule trigger (weekly/monthly)
  3. Google Sheets node to pull data
  4. Code node to format the report
  5. Email node (or MailerLite node) to send
This is the most custom package. Scope it carefully during discovery.
Estimated build time: 2-6 hours depending on complexity

2Day 2[Helpful content]Educational content related to their need (positions client as expert)

  • Website form: MailerLite group via embed code
  • Other sources: Show client how to manually add, or use Zapier/n8n
